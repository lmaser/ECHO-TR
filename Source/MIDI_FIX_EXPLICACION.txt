PROBLEMA ENCONTRADO CON MIDI
=============================

El código NO estaba filtrando por canal MIDI.

ANTES:
------
- Iteraba sobre TODOS los mensajes MIDI sin verificar el canal
- El parámetro "MIDI Port" (1-16) se configuraba pero no se usaba
- Cuando ponías "1", esperabas recibir solo notas del canal 1
- Pero el código procesaba TODOS los canales

SOLUCION APLICADA:
------------------
Ahora en processBlock():

1. Lee el selectedMidiPort (1-16, o 0=deshabilitado)
2. Para cada mensaje MIDI: comprueba msg.getChannel() == selectedMidiPort
3. Si no coincide el canal: lo salta con "continue"
4. Solo procesa notas del canal correcto

CÓDIGO CLAVE:
-------------
```cpp
const int selectedMidiPort = midiPort.load(...); // 1-16

for (const auto metadata : midiMessages) {
    const auto msg = metadata.getMessage();
    
    // FILTRAR POR CANAL MIDI
    if (msg.getChannel() != selectedMidiPort)
        continue; // Saltar mensajes de otros canales
    
    // Procesar nota...
}
```

CÓMO PROBAR:
------------
1. Compila el proyecto
2. En FL Studio:
   - Crea un MIDI Out
   - Configúralo al canal 1
   - Conecta el MIDI Out al plugin
   
3. En el plugin:
   - Activa checkbox MIDI
   - Configura MIDI Port = 1
   
4. Prueba:
   - Toca notas en el MIDI Out
   - El delay time debería cambiar según la nota
   - Nota C4 (60) = 261.6 Hz = tiempo de ~3.82ms

REFERENCIAS JUCE:
-----------------
- MidiMessage::getChannel() devuelve 1-16 (NO 0-15)
- MidiMessage::isNoteOn() verifica velocity > 0
- En VST3, el MIDI se envía vía MidiBuffer en processBlock()
- El wrapper VST3 ya maneja el enrutamiento interno

NOTAS:
------
- Los DBG() solo funcionan en Debug builds con debugger
- Para Release builds, DBG no hace nada
- Por eso no veías ningún output de debug
